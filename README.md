## Задача 1. Процесс разработки (CI/CD)

### Решение
- **GitHub** — хранение кода (Git)
- **GitHub Actions** — CI/CD
- **Docker** — сборка и тестирование
- **Self-hosted runners** — при необходимости запуск агентов на своих серверах

### Обоснование
GitHub Actions позволяет:
- запускать сборку по push/PR и вручную (workflow_dispatch);
- хранить секреты в GitHub Secrets;
- создавать несколько конфигураций сборки (matrix);
- использовать кастомные шаги и Docker-образы;
- запускать сборки и тесты параллельно;
- использовать шаблоны workflow;
- подключать собственные агенты.

Решение полностью облачное и не требует отдельной инфраструктуры.

## Задача 2. Логирование

### Решение
Стек **ELK**:
- Filebeat — сбор логов (stdout)
- Logstash — обработка
- Elasticsearch — хранение
- Kibana — поиск и UI

### Обоснование
- Централизованный сбор логов со всех хостов  
- Приложения просто пишут в stdout  
- Filebeat гарантирует доставку  
- В Kibana есть поиск, фильтрация, сохранённые запросы и ссылки  
- Можно выдавать доступ разработчикам через роли

## Задача 3. Мониторинг

### Решение
- Prometheus — сбор метрик  
- Node Exporter — метрики хостов  
- cAdvisor — метрики контейнеров  
- Grafana — дашборды и визуализация  

### Обоснование
- Собираются метрики CPU, RAM, Disk, Network  
- Есть метрики по каждому сервису  
- Можно добавлять кастомные метрики приложений  
- Grafana позволяет строить запросы и настраивать дашборды  
- Стек является стандартом для микросервисов и Kubernetes
